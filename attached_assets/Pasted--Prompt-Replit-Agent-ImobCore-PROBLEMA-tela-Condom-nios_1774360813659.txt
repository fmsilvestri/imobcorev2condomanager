## FIX 1 — Backend: `src/routes/master.ts`
## GET /condominios retorna [] vazio quando banco vazio — trocar fallback

### Localizar e SUBSTITUIR:
```typescript
// ANTES (linha ~39):
r.get("/condominios", async (_req, res) => {
  try {
    const { data } = await getDB().from("condominios").select("*").order("nome");
    res.json(data ?? getDemoCondos());
  } catch { res.json(getDemoCondos()); }
});
```

### Por:
```typescript
// DEPOIS — fallback quando banco vazio OU erro:
r.get("/condominios", async (_req, res) => {
  try {
    const { data, error } = await getDB().from("condominios").select("*").order("nome");
    // data=[] quando banco vazio — usar demo nesse caso também
    if (error || !data || data.length === 0) {
      return res.json(getDemoCondos());
    }
    res.json(data);
  } catch { res.json(getDemoCondos()); }
});
```

## FIX 2 — Frontend: `public/master/index.html`
## init() carrega CONDOS_DEMO fixo, mas não busca do backend. Corrigir.

### Localizar e SUBSTITUIR a função init():
```javascript
// ANTES:
async function init(){
  S.condos=CONDOS_DEMO;
  S.usuarios=USERS_DEMO;
  // ...
}
```

### Por:
```javascript
async function init(){
  // Tenta carregar do backend primeiro; fallback para DEMO se falhar
  try {
    const [condosRes, usersRes] = await Promise.allSettled([
      api("/master/condominios"),
      api("/master/usuarios"),
    ]);
    S.condos = (condosRes.status==="fulfilled" && Array.isArray(condosRes.value) && condosRes.value.length > 0)
      ? condosRes.value : CONDOS_DEMO;
    S.usuarios = (usersRes.status==="fulfilled" && Array.isArray(usersRes.value) && usersRes.value.length > 0)
      ? usersRes.value : USERS_DEMO;
  } catch {
    S.condos   = CONDOS_DEMO;
    S.usuarios = USERS_DEMO;
  }
  S.regrasCurr=[...REGRAS_DEMO];
  populateCondoSelects();
  await loadStats();

  renderUsuarios();

  renderModSelects();
```

## FLUXO COMPLETO — Criar condomínio + Di personalizada + usuários

### PASSO 1: Master cria o condomínio
**No painel Master → Condomínios → "+ Novo condomínio"**

O endpoint já faz tudo automaticamente:
```
POST /api/master/condominios
{ nome, total_unidades, total_moradores, plano, endereco, cidade }

↓ action no backend (master.ts):
1. INSERT condominios
2. criarConfigDiPadrao(id) → INSERT di_configuracoes com defaults
   { nome_di: "Di", tom: "direto_empatico", limite: 500, di_ativa: true,
     modulos_ativos: ["os","financeiro","iot","misp","encomendas",...] }
```

### PASSO 2: Master personaliza a Di do condomínio
**Master → Config por Condo → selecionar → "Configurações Di"**

```
PUT /api/master/di/settings/:condoId
{ nome_di: "Dircelene", tom_comunicacao: "amigavel",
  limite_financeiro: 1000, di_ativa: true,
  modulos_ativos: ["os","financeiro","iot","comunicados","encomendas"] }
```

**Master → Config por Condo → "System Prompt" → bloco B01 Identidade**
```
PUT /api/master/di/config/condo/:condoId/1_identidade
{ conteudo: "Você é Dircelene, síndica virtual do {{nome_condominio}}.
  Receptiva, empática e pronta para ajudar cada morador.
  Tom: {{tom_comunicacao}}. Limite: R$ {{limite_financeiro}}." }
```

### PASSO 3: Master cria usuários para o condomínio
**Master → Usuários → "+ Novo usuário" → selecionar condo**

```
POST /api/master/usuarios
{ nome, email, role: "gestor"|"sindico"|"morador"|"zelador",
  condominio_id: UUID_DO_CONDO,
  unidade_id: "802"  // só para morador }
```

**Perfis disponíveis:**
| role | o que pode fazer |
|------|-----------------|
| gestor | tudo no painel gestor — OS, financeiro, aprovações Di |
| sindico | comunicados, OS (sem custo), votações |
| morador | seus dados, encomendas, reservas, chat Di |
| zelador | OS operacionais, portaria |

### PASSO 4: Di já está personalizada e ativa
A partir do login de qualquer usuário deste condo:
- Chat Di usa o `nome_di` e o `system prompt` configurados pelo Master
- Módulos visíveis = apenas os marcados em `modulos_ativos`
- Limite financeiro = o configurado em `di_configuracoes`
