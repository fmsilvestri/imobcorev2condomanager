# ImobCore v2 — Condominium Management SaaS

## Overview

ImobCore v2 is a full-stack SaaS platform designed for comprehensive condominium management. It features an AI-powered "Síndico Virtual" (virtual superintendent) named Di, leveraging Anthropic Claude for intelligent automation and interaction. The platform aims to streamline condominium operations, enhance communication, and provide data-driven insights for managers and residents. Key capabilities include multi-condo management, maintenance tracking, financial oversight, IoT integration, and an intelligent concierge system for visitors and residents. The project seeks to revolutionize condominium administration by integrating advanced AI with practical management tools.

## System Architecture

ImobCore v2 is built as a pnpm workspace monorepo using Node.js ESM, Express 5, Supabase, and React with TypeScript.

**UI/UX Decisions:**
- **Frontend:** React + Vite, providing three distinct views:
    - **Painel Gestor:** Comprehensive desktop dashboard with sidebar, KPI cards, AI chat, OS CRUD, financials, IoT sensor data, MISP alerts, maintenance, and parcel tracking.
    - **App Síndico:** Mobile-optimized interface for superintendents with dark/light themes, condo photo management, quick stats, AI integration, and a module grid.
    - **App Morador:** Mobile mockup for residents to view announcements, services, and status updates.
- **Onboarding Wizard:** An 8-step guided process for new condominium setup, including configuration of condominium details, physical structure, residents, IoT sensors, financial parameters, and AI persona customization.
- **Theming:** Dynamic theme application, including color schemes and avatars, for the multi-condo concierge totem.

**Technical Implementations & Design Choices:**
- **Monorepo Structure:** Managed with pnpm workspaces, allowing each package to handle its dependencies independently.
- **Backend API:** Express 5 server providing a RESTful API for all platform functionalities.
- **Database:** Supabase (PostgreSQL) for data persistence, managed via `@supabase/supabase-js`.
- **AI Integration:** Anthropic Claude (claude-sonnet-4-6) is used for the "Síndico Virtual" (Di) for tasks like chat, communication generation, maintenance analysis, financial insights, and employee scheduling.
- **Real-time Communication:** Server-Sent Events (SSE) at `/api/stream` for real-time updates and notifications across the platform.
- **Di Engine:** A core AI engine configured per condominium, allowing for contextual AI interactions across all modules. It uses a flexible system prompt structure stored in Supabase (`di_system_prompt`, `di_configuracoes`) and supports various user profiles (`master`, `gestor`, `sindico`, `morador`, `zelador`).
- **Concierge Virtual Totem:** A dedicated Single Page Application (SPA) for condominium entrances, offering visitor management, deliveries, resident guidance, and emergency contacts. It includes TTS/STT capabilities and is configurable per condominium with unique tokens.
  - **Módulo Hóspede (Guest Module):** Full guest experience screen with 5 tabs: Áreas (2-col area cards with colored icons + detail overlay with horários/regras/dicas), Serviços, Região (tourist points + restaurants), Parceiros (gold partner cards with discount badges), Emergência (large CTA buttons). Detail overlay has slide-up animation. Area cards connect to Di chat. Data loaded from `/api/concierge/guia` endpoint. Rich data stored in `conteudo` JSONB column and merged at read time.
  - **Config fields:** `wifi_rede`, `wifi_senha`, `checkin_h`, `checkout_h`, `cidade` exposed by `/api/concierge/config`. Requires SQL migration: `ALTER TABLE di_configuracoes ADD COLUMN IF NOT EXISTS concierge_wifi_rede TEXT DEFAULT '', concierge_wifi_senha TEXT DEFAULT '', concierge_checkin_h TEXT DEFAULT '14h', concierge_checkout_h TEXT DEFAULT '12h', concierge_cidade TEXT DEFAULT 'Rio de Janeiro';`
- **Maintenance Module:** Manages equipment, maintenance history, preventive plans generated by AI, and alerts.
- **Comunicados Module:** Multi-channel communication system with AI-generated content, template management, scheduling, and metrics, supporting WhatsApp and Telegram.
- **Maintenance Plans Module:** Comprehensive management of maintenance plans, including CRUD operations, sector-based filtering, AI-driven plan generation, and an annual calendar view.
- **Employee & Smart Schedule Module:** Manages staff, AI-generated schedules, briefings, and defines common areas. Includes performance ranking, estimated labor liabilities, and operational risk analysis.
- **Intelligent Financial Import Module:** Supports importing financial data from OFX, PDF, and image files (PNG/JPG) using OCR and AI vision. Features include heuristic classification, automated learning, duplicate detection, and confidence scoring.

## External Dependencies

- **Supabase:** Used as the primary database (PostgreSQL) and for authentication.
- **Anthropic Claude:** AI model used via Replit AI Integrations for the "Síndico Virtual" (Di) functionalities.
- **ElevenLabs:** Text-to-Speech (TTS) service used for the Concierge Totem, with a `web_speech` fallback.
- **Z-API:** (WhatsApp) for sending multi-channel communications.
- **Telegram Bot API:** For sending multi-channel communications.
- **pdf-parse:** Library used for parsing PDF documents in the financial import module.
